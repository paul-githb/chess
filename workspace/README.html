AP Computer Science A (Best) Final Project (Ever)

Project Name: Chess

Team Members: Eric Wu, Yatin Ravi, Paul Shin

Achievements:

  1. Data Structures: This project demonstrates use of data structures by representing the chess board as a two-dimensional, 8x8 array of tiles containing pieces, as well as using two ArrayLists of Pieces to store all captured pieces.

  2. Inheritance: This project demonstrates inheritance as the Piece class takes the role of the parent class, with all other piece types deriving from this one class: Pawn, Rook, Knight, Bishop, King, and Queen.

  3. Sorting: This project demonstrates the use of sorting in the form of insertion sort. This sorting algorithm was put to use in the sorting of the sets of captured pieces, which was defined and maintained in the Player class. The pieces were sorted by their piece value, with the King in the front with a piece value of 0 (useful in determining whether or not a winner was found) and the Knights in front of the Bishops.

  4. Abstraction: This project demonstrates abstraction through the use of declaring abstract methods inside the Piece class, which had to be defined in each individual derived class. These two methods are: boolean moveIsValid(int, int, Board), which took two integer values and determined whether or not moving the piece to that position on the board was valid (disregarding checks, only for validity), and String getStringValue(), which simply returned the String representation of the piece for use in the text-based game. (This was not entirely needed, but good for "demonstrating complexity.")

Some notable features of our project include:

  • Promotion: This is a chess rule which states that if a pawn makes it to the other end of the board, then it must be replaced immediately with one of the following: a queen, knight, rook, or bishop. When a pawn satisfies this rule, a popup message appears in which the player who owns the pawn chooses which piece to replace the pawn with via a drop down menu. Play continues as normal with the opposing player's turn, unless there is a checkmate.

  • Castling: This is a special move in Chess in which the King advances two squares toward any given rook belonging to the same player, and that rook is placed on the opposite side of the King. This move is valid as long as the following conditions hold:

    • The rook has not been moved.
    • The king has not been moved.
    • There exists no piece in between the king and the rook.
    • The move will not put the king under a check.

    The way our game handles this move is by performing it only when the appropriate CastleButton is pressed. There exists four CastleButtons in the frame, and they are only visible when a castle is available (i.e. the above conditions hold). The accessibility of one button does not depend on that of the other buttons.

  • Light Indicator: Each player has a little light indicator saying whether or not it's the player's turn. This makes it easy to determine whose turn it is without having to display text on the screen.

  • Checks: A rule that's integrated in most special rules or moves in Chess is called the check. If a player is under check, then his or her King piece is currently being threatened. If the player under check does not respond appropriately to counter the check, then the player's move is not valid. I.e. the player must attempt to counter the check in order to have a valid move. Possible methods of countering a check include:

    • Moving the King to a tile which does not put it under check.
    • Capturing the piece which is currently threatening the King.
    • Blocking the check using one of the player's own pieces.

    If the player is unable to respond to the check in any way, then the player is under a checkmate. This means that the opposing player has won the game.
